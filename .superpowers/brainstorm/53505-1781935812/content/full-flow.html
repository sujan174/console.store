End-to-end flow · console.store ⟷ Swiggy MCP

Every step maps to a Swiggy tool. COD-only, browser OAuth, 5-day token baked in.

① First run — link account

ssh console.store→ unknown pubkey→ browser OAuth 2.1 + PKCE→ phone + OTP→ token stored (5-day TTL)→ account ready
One-time. Re-prompts only on 401 / every 5 days (no refresh tokens in v1).

② Returning — load menu

ssh console.store→ splash "fetching grub"→ address get_addresses→ nearby search_restaurants→ filter to curated→ MENU

③ Order — Food (one restaurant / cart)

pick place search_restaurants→ items get_restaurant_menu→ ❯ add update_food_cart→ cart get_food_cart→ coupon apply_food_coupon→ place place_food_order · COD→ CONFIRMED (ascii)→ track track_food_order
Switch restaurant → flush_food_cart ("new cart?"). Cap ₹1000. No scheduling.

④ Pay — no in-app step

"Pay ₹338 to rider · cash or UPI on delivery"
v1 = COD only. Online payment arrives in Swiggy API v2+.

⑤ + ⑥ Shortcuts

↵ the usual→ get_food_orders → re-add → checkout retention wedge
snacks (Instamart)→ your_go_to_items / search_products→ update_cart→ checkout · COD separate cart · min ₹99
